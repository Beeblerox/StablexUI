class UIBuilder
Core class. All macro magic lives here
static public var defaults : Hash<Hash<Widget->Void>> = new Hash()
Closures for applaying default settings to widgets. Closures created with UIBuilder.init('defaults.xml')
static public var skins : Hash<Void->Skin> = new Hash()
registered skins
@:macro static public function saveCodeTo (dir:String) : Expr
Set directory to save generated code to. Should be called before .init() If you get any compiler errors on your xml files, you can find corresponding file with generated code to find out what was wrong.
@:macro static public function init(defaultsXmlFile:String = null) : Expr
Initializing UIBuilder. Should be called before using any other UIBuilder methods except .reg* methods @param defaultsXmlFile - path to xml file with default settings for widgets
@:macro static public function buildFn (xmlFile:String) : Expr
Generates closure for widget creation. xmlFile - path to xml file with markup. In xml you can use these placeholders: $this - replaced with current widget; $SomeClass - replaced with com.some.package.SomeClass. If registered with UIBuilder.regClass('com.some.package.SomeClass'); #widgetId - replaced with UIBuilder.get('widgetId'); #SomeClass(widgetId) - replaced with UIBuilder.getAs('widgetId', SomeClass); SomeClass must be of Class<Widget> @someParam - replaced with arguments.someParam. Arguments can be passed like this: UIBuilder.buildFn(xmlFile)({someParam:'some value', someParam2: 3.14}); @throw String if .init() was not called before @throw String if one of used in xml widgets, classes or events was not registered by .regClass() or .regEvent() @return Dynamic->Root_Xml_Element_Class
@:macro static public function regEvent (shortcut:String, eventType:String) : Expr
Register event type to declare event listeners in xml (attributes prefixed with `on-[shortcut]`). @throw String if this shortcut is already used
@:macro static public function regClass (fullyQualifiedName:String) : Expr
Register class to use it in xml code. For example 'com.pack.SomeClass' can be referenced in xml like this: $SomeClass @throw String if fullyQualifiedName is wrong (does not match `com.package.ClassName` notation) @throw String if class is already registered. E.g. com.pack1.MyClass and org.pack2.MyClass can not be registered simultaneously, because both will be shortened to $MyClass for usage in xml. You still can register one of them and use another one by it's full classpath in xml
@:macro static public function regSkins(xmlFile:String) : Expr
Register skin list. See samples/handlers_skinning for example skins.xml @throw String if UIBuilder.init() was not called before @throw String if one of tag names in xml does not match ~/^([a-z0-9_]+):([a-z0-9_]+)$/i @throw String if class specified for skin system is not registered with .regClass
static public function create<T>(cls:Class<T>, properties:Dynamic = null) : Null<T>
Creates widgets at runtime. @param cls - create widget of this class; @param properties - read description of .apply() method below. @throw Dynamic if corresponding properties of `cls` and `properties` have different types @throw String if `cls` is not of Class<Widget>
static public function apply(obj:Dynamic, properties:Dynamic) : Void
Apply properties to object. e. g. propList = { prop1: -2, prop2: true, prop3: { nested1: 'val1', nested2: null, }, } than after calling UIBuilder.apply(someObj, propList) we will get following: someObj.prop1 == -2 someObj.prop2 == true someObj.prop3.nested1 == 'val1' someObj.prop3.nested2 == null Note: non-scalar object properties must not be null, otherwise you'll get an exception "Can't set property of Null" @throw Dynamic if corresponding scalar properties of `obj` and `properties` have different types

Get registered skin

static public inline function getAs<T> (id:String, cls:Class<T>) : Null<T>
Return widget as instance of specified class. If widget is not of that class, returns null
